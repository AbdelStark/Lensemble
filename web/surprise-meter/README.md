# surprise-meter — Milestone-1 live meter with stage fallback

The live **surprise-meter** for the Codex-Paris demo (issue
[#338](https://github.com/AbdelStark/Lensemble/issues/338), Milestone 1). By default the page
renders the recorded, calm stage trajectory and then runs a live ONNX held-out check in the
background when the runtime is available. This keeps the on-stage visual stable: surprise spikes
come from operator perturbation controls, not from an autonomous replay changing modes. The zero
ONNX/network fallback loads `data/result_card.json` and `data/surprise_trajectory.json`, then
renders the certified held-out numbers beside the recorded TwoRooms trajectory.

Design language is **"The Offprint"** — the same warm-bone-paper / oxblood / Fraunces+Newsreader+
Spline-Sans-Mono identity as the Demo-Night deck, so the live tool and the slides read as one
piece. The surprise signal is a registered **ink recorder** (a lab-bench chart on paper), not a
glowing widget.

## Run

```bash
# from the repo root — needs http:// (ES modules), not file://
uv run lensemble demo federated --port 8765
# -> http://127.0.0.1:8765/web/surprise-meter/
```

Useful modes:

- `?engine=auto` - default; recorded stage visual plus a background live ONNX held-out check.
- `?engine=live&ep=wasm` - force the live WASM path and live visual for projector-safe validation.
- `?trajectory=live` - with `engine=auto`, replace the stage visual with the live free-running replay.
- `?engine=fallback` - force the recorded trajectory fallback.

Buttons: **Teleport / Off-distribution action / Push through a wall** inject a surprise event;
the **Pre / Post-federation** toggle drops the in-distribution baseline to the certified level;
**❚❚** pauses. Watch the frame-diff (Fig. B) run high on motion while surprise (Fig. A) stays
flat — and "Off-distribution action" spike surprise *without* motion. That is the whole point.

## Data path

| File | Role |
|---|---|
| `surprise_engine.mjs` | Real scalar surprise contract: `surprise = MSE(predicted_next_latent, actual_next_latent) / 192`; ORT-free self-tested. |
| `app.mjs` | `engine=auto|live|fallback` bootstrap, `?ep=wasm|webgpu` provider selection, live ONNX trajectory build, stable recorded stage visual, and browser held-out pre/post check. |
| `fixtures/adapter_offset.json` | Served fallback copy of the server-produced 12,512-float adapter offset from `scripts/surprise/run_clean_round.py`. |
| `data/result_card.json` | Browser-readable copy of the certified result card generated by `scripts/lewm_surprise_check.py`. |
| `data/surprise_trajectory.json` | Recorded fallback trajectory for rehearsal rung C, capped below 600 steps. |
| `mock/engine.mjs` | Fallback renderer that replays the recorded trajectory and preserves the pre/post toggle and perturbation controls. |

The real runtime contract lives in `surprise_engine.mjs`; `node surprise_selftest.mjs` validates
the warm-up behavior, scalar MSE parity, and live-trajectory builder without ONNX Runtime.

## Numbers & claim discipline

Every on-screen figure mirrors the certified evidence at full precision:
**+12.3% this run · +16.8% mean · +5.4% worst (seed 2)** of held-out next-step prediction error,
from `lewm_tworooms_system_probe.json` / `…_probe_seedsweep.json` through
`docs/evidence/lewm_tworooms_surprise.json`. The pre/post baseline is the certified
`baselineMse 0.0604 → adaptedMse 0.0530` (= +12.3%). In live mode the page also computes a
browser-held-out pre/post check from `collectResidentPairs(seed=991)` and labels it separately
from the certified evidence number. The recorded trajectory is a fallback visualization, not a
second benchmark.

The footer renders the binding non-claims from data (it can't silently drop off-screen): it is
*federated adapter continuation on a frozen checkpoint*, surprise is a *scalar* (not a heatmap),
and there is no DP / secure-aggregation / paper-scale / beats-local-only claim.
